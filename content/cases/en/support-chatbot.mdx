    type: text
    body: "~80% of e-commerce support queries are routine: what's the price, how do you deliver, can I pay in installments, what's the return policy. A human agent answers the same questions dozens of times a day — during business hours only. Customers write at 11pm and get a response the next morning. The solution isn't more agents. It's automating what doesn't require a human."

  - title: "Why CAG, not RAG"
    type: cards
    body: "RAG (vector search) is the right tool for large unstructured documents where you don't know what will be asked. For a support bot, the knowledge is structured and fits in the context window — no chunking or vector search needed."
    items:
      - icon: Brain
        title: "RAG — when knowledge is large and unstructured"
        desc: "Thousands of documents, unknown queries. Chunks into vectors, searches semantically. Right for documentation, knowledge bases, books."
      - icon: Zap
        title: "CAG — when knowledge is structured and bounded"
        desc: "Product catalog, prices, policies — fits in the system prompt. No vector database, no embedding step, no search errors. Simpler, faster, cheaper."
      - icon: Database
        title: "Plus: conversation memory"
        desc: "RAG typically has no memory. This bot saves every exchange to Supabase and loads the full history on each call — the context never resets."
      - icon: Shield
        title: "Plus: no hallucinations from search"
        desc: "Vector search can return the wrong chunk. The system prompt is static and controlled — the bot answers only from what you put in it."

  - title: "Conversation memory in Supabase"
    type: text
    body: "Every exchange is saved as a row: user message and AI response. On each new message, the bot loads all previous rows, formats them into a proper messages array, and passes them to Claude. The bot knows what was discussed earlier in the session — no repeated questions, no lost context."

  - title: "Prompt caching — the cost layer"
    type: text
    screenshot: "/cases/support-chatbot/supabase-memory.png"
    body: "The system prompt contains 1217 tokens of store knowledge. On the first call it's written to cache (cache_creation_input_tokens: 1217). On every subsequent call it's read from cache at 10% of the regular token price. That's 87% cheaper per cached call. For a bot handling hundreds of queries, it adds up fast. The extended-cache-ttl-2025-04-11 beta header extends the default 5-minute TTL to 1 hour — the cache stays warm across the whole working day. The Supabase table shows exactly what gets saved: each exchange as a row, both turns visible."
